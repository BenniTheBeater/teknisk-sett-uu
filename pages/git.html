Git
How does Git work?
Git operates entirely from within a hidden folder called ".git" in your project directory.
This folder contains the complete history of your project, including all changes and branches.
All this in a single hidden folder, this is the most precious folder you have.
How do you use Git?
Using Git involves writing a series of commands in your terminal or through a Git client.
You start by initializing a Git repository in your project directory with
$ 
git 
init
.
Then, you stage your changes using
$ 
git 
add
 and commit them with
$ 
git 
commit
.
Finally, you can share your work with others by pushing to a remote repository using
$ 
git 
push
.
What is a remote repository?
A remote repository is like a place in the cloud for your code, a place in the digital sky.
It's where you share, store, and collaborate on your projects with a global reach.
You can connect a remote repository to a local one using
$ 
git 
remote 
add 
[name] 
[url]
.
What is a commit?
A commit is like a snapshot in time for your currently staged code.
It makes sure your current code gets remembered, and is easy to revert back to if so needed.
It should be noted that you should try to commit as often as possible, even just for minor changes,
and you should always make the commit message short and straight to the point.
What are staged files?
Staged files are files that are currently getting tracked by Git.
You can stage a file with the command
$ 
git 
add 
[path]
.
When staged, the file's changes will be included in the next commit.
