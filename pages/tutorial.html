Tutorial
Project Setup
First of all, you'll need to create a git repository.
You can create a Git repository with the command
$ 
git 
init
.
This will create a hidden ".git" folder in the current directory.
The ".git" folder will contain everything from the commit history to branches.
Committing
When you have made changes that you want to record, you have to commit them.
Before committing, you'll first have to stage your files using the command
$ 
git 
add
 [path]
,
where
[path]
 is the path to the folder or file you want to stage,
this is often just
.
 meaning the current directory along with all its sub-folders.

Once you have staged the files you want to record the changes of,
you can commit the changes using the command
$ 
git 
commit
 -m
 [message]
 .
Pushing
The next logical step when using Git in conjunction with GitHub, is to push the local changes to a remote repository.
You can do this by first linking the local repository to the remote one with the command
$ 
git 
remote 
add 
[name] 
[url]
,
where
[name]
 is a name of your choice to refer to the remote repository (often just "origin") to be used in future Git commands,
and
[url]
 is the URL of the remote repository you want to link to.

Once you have successfully linked a remote repository, you can push your local commits using the command
$ 
git 
push
.
Pulling
When remote changes surpass the local ones, you'll want to pull those changes into your local repository,
to do this, you can use the command
$ 
git 
fetch
 followed by
$ 
git 
pull
.
These commands will effectively merge the remote changes with your local ones,
making your local repository up to date with the remote one.
Branching
The time comes when you'll want to branch out to work on bigger changes by yourself.
You can create a new branch with the command
$ 
git 
branch 
[name]
,
where
[name]
 is the name of the new branch.
To change branches back and forth, you can use the command
$ 
git 
checkout 
[name]
,
where
[name]
 again is the name of the branch you want to go into.

Now, all the changes you make will be contained in the currently active branch,
and will not disturb the other branches until you merge.
Merging
When you have finished working on a branch, you might want to merge those changes into the main branch.
You can merge two branches with the command
$ 
git 
merge 
[name]
,
where
[name]
 is the name of the branch you want to merge into the currently active branch.
Keep in mind that this merges the specified branch into the currently active one,
meaning that you'll want to be in the branch you want to merge into.

It's also worth noting that there is a possibility for merge conflicts,
in which case, you'll need to resolve those before continuing the merge.
You can do this by going into the files provided by Git, and sorting out the wanted and unwanted code,
Then continuing the merge by typing the command once again.
That's all
And that's all you need to know to start using Git.
I hope that this has showed you the general workflow for Git,
and even though not everything was covered extensively, I hope that you at least learned something.
